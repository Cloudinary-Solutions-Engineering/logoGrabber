# Logo Grabber

Logo Grabber is a bookmarklet-powered tool for finding SVG logos on the current web page, previewing them visually, downloading them, or uploading them directly to Cloudinary.

It is designed for quickly capturing inline SVG logos and SVG logo files from websites.

## Features

- Finds inline `<svg>` elements on the current page.
- Finds SVG files used in `<img>`, `<object>`, and `<embed>` elements.
- Finds SVGs referenced through `<use>` and SVG sprite patterns where accessible.
- Searches inside open shadow DOM roots.
- Displays all SVG candidates in a visual picker.
- Sorts candidates by rendered pixel area, largest first.
- Downloads the selected SVG.
- Uploads the selected SVG to Cloudinary using an unsigned upload preset.
- Names downloaded and uploaded assets using the root domain of the current website.
- Lets each user generate a personalised bookmarklet containing their own Cloudinary cloud name and upload preset.

## How the Cloudinary details are set

Cloudinary details are entered on the `index.html` install page before the bookmarklet is added to the browser.

The install page asks for:

1. Cloudinary cloud name
2. Cloudinary unsigned upload preset

Once both fields are completed, the page generates a personalised bookmarklet. That bookmarklet embeds the values into this JavaScript object:

```js
window.LOGO_GRABBER_CONFIG = {
  cloudName: "your-cloud-name",
  uploadPreset: "your-unsigned-upload-preset"
};
```

When the bookmarklet runs on a website, it first sets `window.LOGO_GRABBER_CONFIG`, then loads the hosted `script.js`.

The script should read the config like this:

```js
const CLOUDINARY_CONFIG = window.LOGO_GRABBER_CONFIG || getConfig();
```

This means:

- users enter their details once on the install page
- those details travel with their saved bookmarklet
- the script does not need to prompt separately on every website
- the same hosted `script.js` can be shared by the whole team

The `getConfig()` prompt flow can remain as a fallback for anyone using an old or manually created bookmarklet that does not include `window.LOGO_GRABBER_CONFIG`.

## Important: no folder parameter

The current script does **not** prompt for, store, or send a separate upload folder.

Any folder/path behaviour should be configured directly in the Cloudinary unsigned upload preset.

The upload request sends:

- `file`
- `upload_preset`
- `public_id`
- `tags`

It does **not** send `folder`.

## Installation

Host both files somewhere publicly accessible, such as GitHub Pages:

```text
index.html
script.js
```

Then share the `index.html` page with your team.

Each user should:

1. Open the install page.
2. Enter their Cloudinary cloud name.
3. Enter their unsigned upload preset.
4. Drag the generated **SVG Logo Grabber** button to their bookmarks bar.

The bookmarklet generated by the page will look broadly like this:

```js
javascript:(()=>{window.LOGO_GRABBER_CONFIG={cloudName:"demo",uploadPreset:"logo_grabber"};let s=document.createElement("script");s.src="https://cloudinary-solutions-engineering.github.io/logoGrabber/script.js?"+Date.now();document.body.append(s)})()
```

The actual generated bookmarklet may be minified or encoded slightly differently.

## Usage

1. Visit a website whose SVG logo you want to capture.
2. Click the **SVG Logo Grabber** bookmarklet.
3. Review the SVG candidates in the picker.
4. Choose either:
   - **Download** to save the SVG locally.
   - **Upload** to upload the SVG to Cloudinary.
5. After a successful upload, the Upload button changes to **Open ✓** and opens the uploaded Cloudinary asset.

## Asset naming

Downloaded and uploaded assets are named using the root domain of the website plus `_logo`.

For example:

| Website | Downloaded file name | Cloudinary public ID |
|---|---|---|
| `https://www.example.com` | `example.com_logo.svg` | `example.com_logo` |
| `https://shop.example.co.uk` | `example.co.uk_logo.svg` | `example.co.uk_logo` |
| `https://www.cloudinary.com` | `cloudinary.com_logo.svg` | `cloudinary.com_logo` |

The `.svg` extension is used for local downloads. The Cloudinary `public_id` is sent without the `.svg` extension.

## Candidate ordering

SVG candidates are sorted by rendered pixel area, largest first.

This is intended to push small icons, social media logos, and site furniture lower in the picker. Some large decorative SVGs or illustrations may still appear before the actual logo, so users should visually confirm the correct asset before downloading or uploading.

## Cloudinary upload preset requirements

Use an unsigned upload preset.

Recommended preset configuration:

- Allow SVG uploads if your account/policy requires format restrictions.
- Configure any required target folder/path in the preset.
- Decide whether overwrite should be allowed.
- Add any moderation, review, or transformation rules your team needs.
- Restrict the preset as much as practical for your use case.

Do not put an API key, API secret, or signed upload credentials in the bookmarklet or client-side JavaScript.

## What the script finds

Logo Grabber scans for:

- inline SVG elements
- SVGs loaded through image, object, and embed tags
- SVG symbol and sprite references where accessible
- SVGs inside open shadow DOM roots

## Limitations

Logo Grabber can only access content available to the page’s JavaScript context.

It may not be able to capture:

- SVGs inside closed shadow DOM roots
- SVGs inside cross-origin iframes
- logos rendered to canvas
- CSS background images that reference SVGs
- external SVG files blocked by CORS when attempting upload

Inline SVGs are generally the most reliable candidates because the script can serialise them directly from the DOM.

## Troubleshooting

### The bookmarklet asks me for Cloudinary details again

Check that your `script.js` reads from the bookmarklet config:

```js
const CLOUDINARY_CONFIG = window.LOGO_GRABBER_CONFIG || getConfig();
```

If the script still uses only:

```js
const CLOUDINARY_CONFIG = getConfig();
```

then it will ignore the details embedded in the personalised bookmarklet.

### Upload failed

Possible causes include:

- the upload preset is not unsigned
- the upload preset does not allow SVG files
- the cloud name is incorrect
- the external SVG is blocked by CORS
- the preset has restrictions that reject the upload
- the asset public ID already exists and overwrite is disabled

### No SVGs found

The page may not use SVG logos, or the logo may be rendered as a raster image, CSS background, canvas element, closed shadow DOM element, or inside a cross-origin iframe.

### The wrong SVG appears first

The picker sorts by rendered size, not by semantic logo likelihood. Choose the correct logo visually from the picker.

## Repository structure

A typical repository contains:

```text
README.md
index.html
script.js
```

`README.md` explains setup and usage.

`index.html` is the install page where users enter their Cloudinary details and generate their personalised bookmarklet.

`script.js` contains the logo detection, visual picker, download, and Cloudinary upload logic.
